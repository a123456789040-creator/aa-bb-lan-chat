# Contributing

Thanks for taking the time to contribute.

## Before You Start

- Keep the project small and easy to understand
- Prefer straightforward code over abstract patterns
- Keep the app usable on a trusted local network without extra infrastructure

## Development Flow

1. Install dependencies with `npm install`.
2. Run the app locally with `npm start`.
3. Run `npm test` before submitting changes.
4. Update the README if behavior, setup, or scripts change.

## Code Style

- Use modern CommonJS Node.js for server-side files unless the project is migrated intentionally
- Favor clear names and short functions
- Add comments only when they explain non-obvious behavior
- Keep user-facing copy concise and readable

## Pull Requests

Please include:

- A short summary of the change
- Any setup or testing notes
- Screenshots or terminal output if the UI or developer workflow changed

## Scope

Good contributions:

- UX polish
- Bug fixes
- Better docs
- Lightweight tests
- Small quality-of-life improvements

Please avoid:

- Large framework rewrites
- Heavy dependencies without a clear payoff
- Features that assume deployment to the public internet
